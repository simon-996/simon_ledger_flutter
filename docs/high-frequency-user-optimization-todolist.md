# 高频使用体验优化 TODO

本文档记录从高频用户视角审查 Simon Ledger 后确认的优化项。每完成一项，将 `[ ]` 改为 `[x]`，并单独提交和推送代码。

## 原则

- [ ] 操作优先落地本地，网络请求不得阻塞记账和账本维护。
- [ ] Android、iOS、Web 共用业务逻辑，平台差异集中在适配层。
- [ ] 每项修改同时验证未登录本地模式、登录在线模式、登录离线模式和本地临时账本同步场景。
- [ ] 每项修改完成后更新本文档，并执行对应测试。

## 1. 离线同步可靠性

- [x] 增加统一的本地 UUID 到远端 UUID 映射能力。账本和人员映射由 `SyncIdentityResolver` 统一记录和解析。
- [x] 修复本地临时账本同步后新增人员的同步链路。上传前解析账本 `syncedRemoteUuid`，成功后替换人员 UUID 引用。
- [x] 修复本地临时账本同步后新增流水的同步链路。上传前转换账本 UUID、付款人 UUID 和使用人员 UUID。
- [x] 固定依赖同步顺序：账本 -> 人员 -> 流水，避免流水引用尚未上传的人员。
- [ ] 为本地临时账本同步、离线新增人员、离线新增流水、恢复网络自动同步补充完整测试。

## 2. 缓存先行加载

- [x] 账本列表改为立即读取本地缓存，后台拉取云端数据后刷新 UI。无网时不等待网络超时。
- [x] 人员列表改为立即读取本地缓存，后台拉取云端数据后刷新。
- [x] 流水列表保持本地缓存先行，后台同步并刷新云端数据。
- [x] 统计首页优先基于本地缓存计算，后台只刷新需要的账本。
- [x] 记账保存时创建者信息优先读取本地资料，不等待 `/api/auth/me` 网络请求。

## 3. 统一同步协调

- [x] 新增 `SyncCoordinator`，先集中协调账本、人员和流水同步。账户资料继续沿用独立防抖同步服务。
- [ ] 增加同步触发策略：App 启动、进入账本列表、进入账本详情、用户手动同步、网络恢复。
- [ ] 避免重复并发同步同一数据，支持失败退避和后续重试。
- [ ] 增加同步中心：展示待同步账本、人员、流水数量，以及最后同步时间。
- [ ] 同步失败时展示用户可理解的提示，并提供重试入口。

## 4. 账本列表体验

- [ ] 云端账本列表直接使用批量人员缓存，避免每张卡片再次触发单账本人员请求。
- [ ] 本地临时账本同步成功后与云端账本合并展示，避免同名账本出现两份。
- [ ] 登录后允许拖动账本排序。第一阶段保存为本机偏好，不阻塞云端使用。
- [ ] 账本卡片统一展示账本、人员、流水的同步摘要，不只提示流水同步状态。

## 5. 快速记账体验

- [x] 记账成功动画支持点击屏幕立即关闭，并避免自动关闭定时器误退出当前页面。
- [ ] 记账保存按钮增加防重复提交状态，避免连续点击生成重复流水。
- [ ] 人员缓存可用时立即允许记账，后台人员刷新不得阻塞保存按钮。
- [ ] 对待同步流水提供轻量状态提示，不打断下一次记账。

说明：不增加“连续记账模式”。保留当前记账流程，只优化响应速度和反馈方式。

## 6. 统计体验

- [ ] 缓存统计页最后选择的账本、时间范围、收支类型和展示币种。
- [ ] 统计页切换账本时优先展示本地结果，后台刷新后平滑更新。
- [ ] 数据较多时减少重复全量加载，避免每次进入统计页面逐账本请求全部流水。

## 7. 邀请与分享

- [ ] 邀请账本支持复制完整邀请文本，不只复制邀请码。
- [ ] 增加可分享的邀请链接，打开后自动填充邀请码。
- [ ] Android 和 iOS 使用系统分享入口，Web 使用复制链接。
- [ ] 保留账本图片分享的平台适配：移动端保存相册，Web 下载 PNG。

## 8. 冲突与恢复

- [ ] 流水版本冲突时展示明确提示，不只显示通用失败信息。
- [ ] 为冲突流水提供刷新云端版本、保留本地副本或重新提交入口。
- [ ] 删除账本、删除人员和删除流水的离线状态统一纳入同步中心。
